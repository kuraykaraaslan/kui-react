## M2 — Interactions (current)

### Goal

Clicking an event opens an anchored popover with title/time/description and Edit/Delete actions. Dragging an empty time-grid range creates an event; dragging a timed event moves it (snap to `slotMinutes`); dragging the bottom edge resizes it. All CRUD goes through new `onEventCreate / onEventUpdate / onEventDelete` props.

### Architecture

**Per-instance Zustand store** — `store.ts`, mirroring [modules/app/RichTextEditor/store.ts](../RichTextEditor/store.ts).

```ts
type CalendarStore = {
  // Navigation (moved out of index.tsx useState)
  date: Date;
  view: View;
  setDate: (d: Date) => void;
  setView: (v: View) => void;

  // Popover
  popover: { event: Event | null; anchorRect: DOMRect | null };
  openPopover: (e: Event, rect: DOMRect) => void;
  closePopover: () => void;

  // Drag preview (single source of truth for ghost rendering)
  drag:
    | { kind: 'idle' }
    | { kind: 'move'; eventId: string; ghostStart: Date; ghostEnd: Date }
    | { kind: 'resize'; eventId: string; ghostEnd: Date }
    | { kind: 'create'; ghostStart: Date; ghostEnd: Date };
  setDrag: (d: CalendarStore['drag']) => void;
};
```

One store per `<Calendar>` mount (via `useRef`), provided through `CalendarStoreProvider`; sub-components read with `useCalStore(selector)`.

### New props

```ts
type CalendarProps = {
  // ...existing...
  onEventCreate?: (range: { start: Date; end: Date }) => void | Promise<void>;
  onEventUpdate?: (event: Event) => void | Promise<void>;
  onEventDelete?: (id: string) => void | Promise<void>;
};
```

Telemetry adds `event-create`, `event-update`, `event-delete` variants.

### Reuse / dependency budget

- `zustand` — already a dep (RTE uses it).
- `cn`, `EVENT_COLOR_CLASSES`, `resolveColor`, `minutesIntoDay`, `addDays`, `startOfDay`, `fmtTime`, `fmtTimeRange` — reuse from existing module files.
- **No new npm packages.** No `floating-ui` (custom viewport-edge flip), no `dnd-kit` (raw pointer events), no `date-fns`. Keep the M1 under-25 kB perf budget.

### Verification

1. `npm run build` (auto-runs `registry:snapshot` via `prebuild`).
2. `npm run dev` → `/showcase` → Calendar entry — exercise all four variants:
   - Month TR — click event → popover anchored to pill; ESC + outside click close it.
   - Week TR — drag "Standup" 9:30 → 14:00; ghost during drag; `onEventUpdate` log.
   - Day EN — grab bottom edge of timed event; height grows in `slotMinutes` increments.
   - Interactive — drag empty 11:00–12:30 range; `onEventCreate` log.
3. Keyboard regression: PageUp/PageDown/T still navigate; ESC during drag cancels.
4. Multi-instance: two `<Calendar>` side-by-side — popover/drag in one does not affect the other.
5. A11y: popover has `role="dialog"`, focus restored on close.

## M3 — RRULE recurrence (deferred)

- Lazy-import `rrule` inside `useRecurrence` (per-visible-range expansion).
- Extend `Event` with `rrule?: string` + `exceptions?: Date[]`.
- Materialise occurrences for the current visible date range only.

## M4 — Resources + multi-calendar overlay (deferred)

- `ResourceView` — one column per `Resource` in the time grid; conflict detection.
- `calendars[]` prop — overlay with per-calendar color + visibility toggles.

## M5 — Agenda + Mini calendar (deferred)

- `AgendaView` — grouped-by-day virtualised list with search + filters.
- `MiniCalendar` — sidebar month grid that jumps the main view to the picked date.

## M6 — A11y / i18n / perf polish (deferred)

- Full WAI-ARIA grid pattern, screen-reader announcements ("Tuesday March 5, 2 events").
- IANA timezone support.
- Virtualised event rendering for high-density weeks.
